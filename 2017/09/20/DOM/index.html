      DOM（文档对象模型）在js编写代码的过程中几乎是不可缺少的。通过DOM，我们可以取得对于网页上的内容，样式进行修改。也通过DOM一系列方法的操作，对于网页中的某一部分进行增删改查。

DOM概览

      DOM是表示和操作HTML和XML文档内容的基础API。DOM并不是很复杂，只是需要理解大量的架构细节。在网页中，一个DOM的结构是树状的，这棵树的最顶层为Document，最底层基本为文本节点。

      这DOM树中每个长方形表示文档的一个节点它表示一个Node对象。接下来就先说这个Node对象。

Node对象

      在上面说过：这DOM树中每个长方形表示文档的一个节点它表示一个Node对象，JS中所有节点类型都继承一Node类型，所以所有节点都有着相同的属性和方法。

  1. nodeType属性，通过节点访问nodeType这个属性可以表明节点的类型，对应的类型nodeType会返回对应的数字（常用的）：
    • 元素节点：1
    • 属性节点：2
    • 文本节点：3
  1. nodeName和nodeValue属性，通过这两个属性可以获得节点的名称和节点的内容。不同的节点类型所获得的nodeName和nodeValue的值是不一样的。


      节点之间的关系也是Node对象中非常重要的关系。节点之间的关系就像传统的家族关系一样，拥有父节点，子节点，兄弟节点。

  1. childNodes属性：每个节点都拥有一个childNodes属性，通过这个属性返回一个NodeList对象（一个类数组对象），来存放该节点下的子节点
  2. parentNode属性：同样每个节点也拥有一个parentNode,通过它的父节点。没有父节点就返回null。
  3. nextSibling和previousSibling属性：通过此属性可以获得他的节点的兄弟属性。previousSibling表示前一个节点性，nextSibling表示后面一个节点.

节点关系:

      在DOM中，有一些非常重要的方法，通过这些方法，我们可以完成对于节点的增删改查.

  1. appendChild():这个方法是一个非常常用的方法，通过它，我们可以在节点的 childNodes列表后插入一个节点，返回新增的节点。
  2. insertBefore():这个方法的用途食欲appendChild()相同的，都是插入，但是insertBefore()的插入比appendChild()更加厉害，他是指插入到特定的位置，方法接受两个参数：一个是要插入的元素，第二个是作为参照的节点。插入后成为参照节点的前一个兄弟节点。
  3. replaceChild()：这个方法是替代某个节点。这个方法接受两个参数：一是要插入的节点，第二个是要替换的节点。替换后插入的节点会到被替换的节点的位子，被替换的节点被移除。
  4. removeChild()：这个方法是移除某个节点。接受一个参数就是他要移除的节点。
  5. cloneNode()：创建某个节点的副本，接受一个参数为Boolean值，表示是否要执行深复制，true为要执行，反之不要。复制后节点归文档所有，如需要，使用上面的方法进行操作。
    • 深复制：复制节点以及他的整个子节点
    • 浅复制: 只复制该节点
  6. normalize()：处理文档中的文本节点。在某个节点上使用时，想后代查找，遇到空文本节点就删除他，找到相邻的文本节点，就合并成一个

Document对象

      Document表示文档，浏览器打开一个 HTML 文档时，该文档就成了一个 document 对象，Document 对象使我们可以对 HTML 页面中的所有元素进行访问。它是window对象的一部分，可用window.document，往往省略window。
